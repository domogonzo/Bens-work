Install Microsoft .NET Framework 3.5 SP 1

If you are using Windows 8, Windows 8.1, Windows Server 2012, or Windows Server 2012 R2, you must install .NET Framework 3.5 SP1.

Depending on your access to the Internet, select one of the following:

  • If you have an Internet connection, install .NET Framework 3.5 SP1 via the Control Panel. For details see: http://msdn.microsoft.com/en-us/library/hh506443.aspx.
  • If you do not have an Internet connection, install .NET Framework 3.5 SP1 with the Deployment Image Servicing and Management (DISM) command-line tool. You will need the original Windows installation DVD and administrative rights to the operating system. Follow these steps:
    1. Insert the Windows installation DVD.
    2. On the Taskbar, click the Start button.
    3. From the Start menu, in the search box, type CMD.
    4. On the Taskbar, right-click the CMD icon, and select Run as administrator.
    5. In the Command Prompt window, enter the following command where <Drive> is the directory location of the Windows installation DVD (for example: ...E:):

      DISM /Online /Enable-Feature /FeatureName:NetFx3 /All /LimitAccess /Source:<Drive>:\sources\sxs

When the installation is completed, The operation completed successfully displays in the Command Prompt window.

For more information on installing .NET 3.5 SP1 on Windows 8 and later, visit: http://support.microsoft.com/kb/2734782

Troubleshooting: Error code 0x800F081F

This error occurs when an alternative installation source is specified and one of the following conditions occurs:

  • The location specified by the path does not contain the installation files.
  • The user does not have READ access to the files.
  • The set of installation files is corrupted, incomplete, or invalid for the version of Windows that you are running.

To resolve this problem, verify that the full path of the source is correct (x:\sources\sxs) and that you have READ access to the location. Try to access the source directly from the affected computer. Verify that the installation source contains a valid and complete set of files. If the problem persists, try a different installation source.
